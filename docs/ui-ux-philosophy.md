# UI/UX 思想

# 詳細UI/UX設計

## 全体設計原則

* 操作は**最短2タップ以内**
* 文字より**数字を主役**
* “判断”をユーザーにさせない（選択肢は極小）
* 毎日使う前提 → **疲れないUI**
* 感情演出は最小（過度な励まし禁止）

---

# 画面構成

## 1. ホーム（中核画面）

**目的：今日を記録させる / 継続を可視化する**

### レイアウト構造（上→下）

#### (A) 現在ストリーク（最重要）

```
37
days smoke-free
```

* 数字は画面最大サイズ
* 1秒で理解できる設計
* タップ不可（誤操作防止）

---

#### (B) 今日の状態入力（最短操作）

```
How was today?

[  今日吸っていない  ]  ← Primary（成功）
[  今日吸った        ]  ← Secondary（再発）
```

UX仕様：

* タップした瞬間に状態更新（確認ダイアログなし）
* **同日内は再入力可能（訂正）**
* ただし「再発 → 成功」に変更時のみ注意表示：

  > “今日吸った記録があります。本当に変更しますか？”

---

#### (C) サマリー（自己効力感ゾーン）

```
Best streak      52 days
Total smoke-free 128 days
Relapses         3
```

UX：

* 数字は中サイズ
* 横並び or カード形式
* タップで履歴へ遷移（自然導線）

---

#### (D) 小さな状態ヒント（非強制）

```
Day resets at 4:00 AM
```

* 設定した日境界を常に表示
* ユーザーの“混乱”を防ぐ

---

### ホームUXルール

* 起動→即ホーム
* 今日未入力なら**自動で(B)に視線誘導**
* 既に入力済ならボタンは**状態表示に変化**

  ```
  ✔ Recorded: Smoke-free today
  [ Change ]
  ```

---

## 2. 再発入力フロー（最短設計）

**トリガー入力は“軽く”する。深掘りさせない**

### 表示条件

* 「今日吸った」押下時のみ表示

### UI

```
What triggered it? (optional)

[ Stress ]
[ Social / Drinking ]
[ Habit ]
[ Urge ]
[ Work ]
[ Boredom ]
[ Other ]

[ Skip ]
```

UX仕様：

* 1タップで確定 → 自動ホーム戻り
* 自由入力は「Other」選択時のみ1行
* **“失敗”という表現は禁止**
* 責めない / 励まさない

---

## 3. 履歴画面

**目的：自己分析・積み上げの可視化**

### 表示

日付降順リスト：

```
Jan 12  ✔ Smoke-free
Jan 11  ✔ Smoke-free
Jan 10  ✖ Relapse — Stress
Jan 09  ✔ Smoke-free
```

UX仕様：

* スクロール高速
* タップで詳細編集可（状態変更・理由変更）
* 空白日は表示しない（ノイズ排除）

---

## 4. 設定画面

**思考停止で変更できるシンプル設計**

### セクション

#### Day Boundary

```
Day resets at:
[ 04:00 AM ▼ ]
```

* 30分刻み選択
* 変更時：今日の計算に影響が出る旨の注意

---

#### Challenge

```
Started date (optional)
[ Jan 1, 2026 ]

Reset all data
```

---

#### Data

```
Export data (JSON)
```

---

## 5. 共有カード生成（拡散導線）

**SNSで自然流入を作る**

### UI（ホームから）

```
[ Share progress ]
```

### 出力内容

```
Smoke-free: 37 days
Best: 52
Total: 128
Relapses: 3
```

UX：

* 画像保存 → OS共有
* 自動投稿UIは禁止（押し付けない）

# マイクロUX設計（継続率を左右）

## 1. 今日未入力の扱い

起動時：

```
You haven’t recorded today
```

※ 赤くしない。プレッシャーを与えない。

---

## 2. 再発後の表示（重要）

ストリーク = 0 のとき：

```
0
New start today
```

※ “失敗” “残念” 等は禁止
※ 目的は「再開の摩擦をゼロにする」

---

## 3. ストリーク更新アニメーション

成功入力時：

* 数字が1だけ増える
* 小さく「+1 day」

※ 過剰演出禁止

---

## 4. 誤操作防止

* ボタンは左右に離す
* 色だけで意味を持たせない（アクセシビリティ）

---

## 5. 習慣化トリガー

ホームに固定表示：

```
Current streak is built one day at a time
```

※ メッセージは1種類のみ（変化させない）

---

# 状態遷移UX

## 今日未入力 → 成功

→ streak +1
→ total +1

## 今日未入力 → 再発

→ streak = 0
→ relapse +1

## 成功 → 再発（同日修正）

→ streak recalculated
→ relapse +1

## 再発 → 成功（同日修正）

→ 確認ダイアログ
→ streak recalculated

---

# UXで絶対やらないこと

* 毎日通知を連打
* モチベ動画 / 名言 / 医療情報
* グラフ過多
* SNS共有を煽る
* 複数禁煙目標（MVP段階）

---

# このUXの狙い（戦略）

* **開く→1タップ→閉じる** が成立 → 継続率↑
* 再発時の心理的摩擦を最小化 → 離脱↓
* 数字中心 → 自己効力感↑
* 思考不要 → 習慣化しやすい
